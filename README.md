# Как запустить проект?

```bash
npm install
npm run dev
```
## Если у Вас уже запущен сервер на порту 3001 
```bash
npm run dev:frontend
```

# Обоснование технологий
1) Vite - для быстрой сборки приложения и моментальной пересборки приложения при помощи Hot Module Reload
2) Jest - простой фреймворк для покрытия кода тестами
3) Redux Toolkit + React Redux - Для упрощения работы с глобальным стейтменеджером. избавляет меня от шаблонного кода и делает редьюсеры компактными. React Redux связывает store с компонентами, позволяя удобно использовать useSelector и useDispatch
4) Typescript - добавление типизации, чтобы избежать ошибок с неверными типами 

# Структура проекта

/
|-- backend # сервер, на который летят запросы
|-- frontend # само приложение



### Внутри папки frontend
```
src/
|-- entry.jsx # Точка входа в приложения. Определяются все пути внутри этого файла
|-- components/
|  |-- item/ # Компоненты карточки объявления и стили для них
|   |-- list/ # Компоненты списка объявлений и стили для них
|   |-- Stats/ # Компоненты статистики модератора и стили для них
|       |-- styles/ # Стили для графиков и блоков статистики
|-- pages/  # Страницы: ItemPage, Stats и др.
|   |-- actions/ # Обертки для API-запросов (чтобы оптимизировать запросы и сохранение ответов в state и localStorage) + вспомогательные функции, которые переиспользуются в компонентах
|-- serverCalls/  # API-запросы к серверу
|-- store/  # Redux Toolkit: slices, entry, кастомные хуки
```

### Какой доп. функционал реализован?
1) Горячие клавишы
2) Сохранение наборов фильтров
3) Темная тема
4) Автообновления. При каждом возвращение на главную страницу обновляются объявления
5) Фильтры по периоду
